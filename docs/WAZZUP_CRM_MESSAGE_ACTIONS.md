# Действия над сообщениями в WhatsApp CRM

## Реализованный UX

- **Mobile**: долгое нажатие (long press ~400 ms) по сообщению включает режим выбора, выделяет сообщение, показывает верхнюю панель действий и панель быстрых реакций над полем ввода. Кнопка «Ещё» открывает bottom sheet с полным списком действий.
- **Desktop**: правый клик по сообщению открывает контекстное меню рядом с курсором.
- **Режим выбора**: можно выбрать несколько сообщений (tap по сообщению в режиме выбора переключает выделение). Reply доступен только при одном выбранном сообщении.
- **Кнопка «Назад» (Android)**: сначала закрывает режим выбора / контекстное меню / диалог пересылки, затем возврат к списку чатов.

## Что работает только в CRM (не синкается в WhatsApp)

| Действие | Описание |
|----------|----------|
| **Удаление** | Мягкое удаление в CRM (`deleted: true`). В WhatsApp сообщение не удаляется. |
| **В избранное (Star)** | Поле `starred` в документе сообщения в Firestore. Только в CRM. |
| **Реакции** | Массив `reactions` в документе сообщения. Только в CRM. |
| **Переслать** | CRM-forward: контент (текст/медиа) отправляется как новое исходящее сообщение в выбранный чат через тот же Send API. В чате помечается как «Переслано» и `forwarded: true` в БД. В WhatsApp это не нативная пересылка. |

## Что сохраняется и в БД, и при отправке в WhatsApp

| Действие | Описание |
|----------|----------|
| **Ответ (Reply)** | При отправке в API передаётся `repliedToMessageId`; в документе сообщения сохраняется ссылка на сообщение в CRM. Отображение цитаты и превью — в CRM. Wazzup/WhatsApp API может не поддерживать reply threading — тогда в самом мессенджере цепочка не отображается. |

## Через API Wazzup

- **Отправка текста/медиа**: поддерживается (в т.ч. с `repliedToMessageId` и `forwarded` в теле запроса — поля сохраняются в CRM).
- **Удаление для всех / редактирование отправленного**: не реализовано (Wazzup API не используется для этого). В CRM только локальное мягкое удаление.
- **Реакции / избранное в WhatsApp**: не синкаются; только в CRM.

## Структура данных (Firestore, коллекция сообщений)

- `deleted`, `deletedAt`, `deletedBy` — мягкое удаление в CRM.
- `repliedToMessageId` — ID сообщения в CRM, на которое отвечают.
- `forwarded` — помечено как пересланное в CRM.
- `starred`, `starredAt`, `starredBy` — избранное в CRM.
- `reactions` — массив `{ emoji, authorId, createdAt }` — только в CRM.
