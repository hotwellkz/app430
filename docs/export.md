# Export MVP (Sprint 11)

## Scope

- Единый export-layer: `@2wix/export-engine`.
- Поддерживаемые форматы:
  - PDF report
  - CSV
  - XLSX

## Pipeline

- API берёт текущую сохраненную версию проекта (`current-version`).
- Строит:
  - panelization snapshot;
  - spec snapshot;
  - export package snapshot.
- Export metadata и snapshot сохраняются в Firestore (`sipEditor_projectExports`).
- UI скачивает файл, используя unified export tables из `export-engine`.

## API endpoints

- `POST /api/projects/:projectId/exports`
- `GET /api/projects/:projectId/exports`
- `GET /api/projects/:projectId/exports/:exportId`

## PDF structure (MVP)

- project title / project id
- version id / version number
- generated at / generated by
- summary (floors, walls, panels, trimmed, area, warnings)
- aggregated BOM
- warnings section

## CSV/XLSX structure

- CSV:
  - aggregated BOM table
- XLSX:
  - `Summary`
  - `BOM`
  - `Walls`
  - `Warnings`

## Version traceability

Каждый export artifact содержит:

- `projectId`
- `versionId`
- `format`
- `createdAt`
- `createdBy`
- `status`
- `fileName`

## Current limitations

- Файлы генерируются на клиенте (browser download fallback).
- `storagePath` пока не используется для object storage.
- Нет async queue и heavy background workers.
- Нет ERP/CNC/advanced production exports.
